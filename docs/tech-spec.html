1. Overview

Stler Tasks is a personal task manager Progressive Web App (PWA). All user data lives in the user's own Google Spreadsheet named db_tasks, which acts as a zero-cost serverless database. The app is fully offline-capable: edits queue locally in IndexedDB and sync to Sheets when connectivity is restored.

Key Design Decisions

DecisionRationale
Google Sheets as databaseNo backend to operate. Data is always owned by the user. Free API quota is sufficient for personal use.
Offline-first writesEvery write goes to IndexedDB immediately. Sync is fire-and-forget via an async queue, so the UI is never blocked by network.
Optimistic UIIn-memory Zustand state updates instantly. Conflict resolution: last-write-wins by updated_at timestamp.
Token in localStorageSingle-user personal tool. XSS tradeoff is explicitly accepted to enable seamless page refresh without OAuth popup.
No URL routerSingle-page app; view state lives in Zustand uiStore. No deeplinks needed for a personal tool.
PWA + service workerInstallable on desktop and mobile. NetworkFirst caching for Sheets API (10 s timeout). Works offline with cached data.

3. Architecture

Pattern: Feature-sliced React SPA — no routing library. All navigation is Zustand state. No backend.

│                        React UI Layer                        │
│  LoginPage · AppShell · Sidebar · TaskList · TaskItem · …   │
└────────────────────┬─────────────────────────────────────────┘
                     │ reads / dispatches
┌────────────────────▼─────────────────────────────────────────┐
│                      Zustand Stores                          │
│  authStore · tasksStore · foldersStore · labelsStore         │
│  uiStore · prefsStore · syncStore                            │
└──────┬──────────────────────┬────────────────────────────────┘
       │ persist (Dexie)      │ enqueue (offlineQueue)
┌──────▼──────┐        ┌──────▼──────────────────────────────┐
│  Dexie DB   │        │           syncService                │
│ (IndexedDB) │        │  scheduleFlush · flush · pull        │
└─────────────┘        └──────┬──────────────────────────────┘
                               │ HTTP REST
                  ┌────────────▼──────────────────────────────┐
                  │  Google Sheets API v4 + Drive API v3       │
                  │  spreadsheet: db_tasks                     │
                  └───────────────────────────────────────────┘

Write Path

  1. User action → tasksStore.addTask() / updateTask() / etc.
  2. Store writes to Dexie immediately (synchronous from user's perspective).
  3. Store calls enqueue('task', 'update', id, payload) → Dexie queue table.
  4. Store calls scheduleFlush() → debounced 800 ms timer.
  5. Timer fires → flush() sends deduplicated queue to Sheets API.

Read Path

  1. On app init: initialLoad() → flush() → pull().
  2. pull() fetches tasks/folders/labels in parallel from Sheets.
  3. Each store's upsertMany() merges into Dexie + memory (last-write-wins by updated_at).
  4. On network reconnect: fullSync() = flush() + pull().
  5. On app returning to foreground (stale > 5 min): fullSync().

Error Handling

ScenarioHandling
Sheets API 401One silent token refresh + retry request
Queue item failsretryCount++; max 5 retries; silently dropped after
Init failure (no network)Fallback to Dexie-only; syncError shown in sidebar footer

7. Authentication & First-Launch Setup

Flow: OAuth 2.0 implicit / token flow via Google Identity Services (GIS).

Scopes: email  ·  profile  ·  https://www.googleapis.com/auth/spreadsheets  ·  https://www.googleapis.com/auth/drive.metadata.readonly

Step-by-Step Auth Flow

1. App.tsx mounts → initAuth()
   ├─ Load GIS script (accounts.google.com/gsi/client)
   ├─ google.accounts.oauth2.initTokenClient({ client_id, scope, login_hint, callback })
   └─ setTokenClient(client)

2. Session restore check (silent, no UI):
   ├─ localStorage token valid (now < tokenExpiry − 60 s)?
   │   └─ Restore session immediately → render AppShell
   ├─ Token expired but user known?
   │   └─ refreshToken() → tokenClient.requestAccessToken({ prompt: '' })
   │       ├─ GIS grants silently → callback → resolveTokenRequest() → AppShell
   │       └─ GIS fails → show LoginPage
   └─ No user → show LoginPage

3. User clicks "Sign in with Google":
   └─ tokenClient.requestAccessToken({ prompt: '' })
       └─ Google consent dialog (first time only) → callback fires

4. GIS callback:
   ├─ Error → rejectTokenRequest(error)
   └─ Success:
       ├─ setToken(access_token, expires_in)
       │   token stored: accessToken, tokenExpiry = now + expiresIn×1000
       └─ fetch /oauth2/v3/userinfo → setUser({ name, email, picture })

5. isAuthenticated = true → AppShell renders

6. AppShell.useEffect():
   ├─ ensureSpreadsheet()   find or create "db_tasks" in Drive
   ├─ initialLoad()         ensure headers; flush queue; pull; ensureInbox
   └─ prefsStore.load()     fetch settings!A1

7. useSync() hook:
   └─ Registers: online/offline/pagehide/visibilitychange listeners

Token Refresh

Proactive: Before every Sheets API call — if Date.now() > tokenExpiry − 60_000, call refreshToken() first.

Reactive: On HTTP 401 response — one silent refresh + retry request.

Logout

Revoke token via google.accounts.oauth2.revoke(token). Clear all Zustand state + localStorage key auth-storage.

First-Launch Spreadsheet Setup

  1. If authStore.spreadsheetId is non-empty → done.
  2. Drive API: search for name='db_tasks' AND mimeType='application/vnd.google-apps.spreadsheet'.
  3. Found: save first result's ID to authStore.
  4. Not found: POST /spreadsheets with body declaring 4 sheets (tasks, folders, labels, settings).
  5. Save new spreadsheet ID to authStore.setSpreadsheetId().

8. Synchronization / API Layer

Sheets API Client (sheetsClient.ts)

Base URL: https://sheets.googleapis.com/v4/spreadsheets/{spreadsheetId}/

Row-index cache: A Map<entityId, rowNumber> avoids re-fetching column A on every update. Invalidated after every flush(). findRowIndex(sheet, id) fetches {sheet}!A:A, scans for the ID, returns 1-based row number or null (falls back to append).

Offline Queue

enqueue(entityType, operationType, entityId, payload):
  db.queue.add({
    entityType, operationType, entityId, payload,
    createdAt: now(), status: 'pending', retryCount: 0
  })

getPending():
  db.queue.where('status').anyOf(['pending','failed'])
          .and(item => item.retryCount < 5).toArray()

markProcessing(localId)  → update { status: 'processing' }
markDone(localId)        → delete record
markFailed(localId, n)   → update { status: 'failed', retryCount: n }
getQueueLength()         → count pending + failed

flush()

1. items = getPending()
2. if items.length === 0: return
3. Deduplicate: Map<"entityType:entityId:operationType" → latest by createdAt>
4. For superseded items: markDone(localId)
5. For each latest item:
   a. markProcessing(localId)
   b. processQueueItem(item):
      task  create → appendTask
      task  update → apiUpdateTask
      task  delete → apiUpdateTask({ ...task, status: 'deleted' })
      folder create → appendFolder  |  folder update → apiUpdateFolder
      label  create → appendLabel   |  label  update → apiUpdateLabel
   c. markDone(localId)                  // success
   d. on error: markFailed(localId, n+1) // will retry up to 5×
6. invalidateRowCache()
7. syncStore.setPendingCount(getQueueLength())

pull()

[tasks, folders, labels] = await Promise.all([
  fetchAllTasks(), fetchAllFolders(), fetchAllLabels()
])
await Promise.all([
  tasksStore.upsertMany(tasks),
  foldersStore.upsertMany(folders),
  labelsStore.upsertMany(labels),
])
syncStore.setLastSyncAt(now())
Conflict resolution (upsertMany): last-write-wins by updated_at. Remote entity replaces local only if remote updated_at is newer. Implemented as a filtered bulkPut.

15. Key Algorithms

ID Generation

generateId(prefix):
  random = crypto.randomUUID().replace(/-/g, '').slice(0, 8)
  return `${prefix}_${random}`
  // → 'tsk_a3f8c912'  |  'fld_9b2e1d7c'  |  'lbl_4f03ab21'

Deadline Status Classification

getDeadlineStatus(deadlineDate, deadlineTime?):
  date  = parseISO(deadlineDate)
  today = startOfDay(now)
  if date < today:           return 'overdue'
  if isToday(date):
    if deadlineTime:
      deadline = today + HH:MM
      if deadline < now:     return 'overdue'
    return 'today'
  if isTomorrow(date):       return 'tomorrow'
  diff = differenceInCalendarDays(date, today)
  if diff <= 7:              return 'week'
  return 'future'

Upcoming Group Builder

useUpcomingGroups():
  pending = tasks.filter(status='pending' AND deadline_date ≠ '')
  sorted  = pending.sortBy(deadline_date ASC)
  today   = startOfDay(now)
  groups  = Map<key, TaskGroup>

  for task in sorted:
    isOver = parseISO(task.deadline_date) < today
    key    = isOver ? 'overdue' : task.deadline_date
    if key not in groups:
      groups[key] = {
        label:     isOver ? 'Overdue' : formatDayGroupLabel(key),
        isOverdue: isOver,
        isToday:   !isOver AND key === todayStr,
        isTomorrow: !isOver AND isTomorrow(parseISO(key)),
        tasks: []
      }
    groups[key].tasks.push(task)

  result = sortBy(groups.values(), isOverdue DESC, key ASC)
  for group in result:
    group.tasks.sortBy(deadline_time OR '99:99' ASC)
  return result

Recurring Task Completion

handleComplete(task):
  if task.is_recurring AND task.deadline_date:
    nextDate = getNextDueDate(task)
    // getNextDueDate: addDays / addWeeks / addMonths by recur_value
    if nextDate:
      updateTask(task.id, { deadline_date: nextDate })
      flush()    // immediate — not debounced
  else:
    completeTask(task.id)
    // completeTask sets status='completed', completed_at=now()
    // also completes all pending subtasks recursively

Flush Deduplication

flush():
  items = getPending()    // status in ['pending','failed'], retryCount < 5
  if items.length === 0: return

  latestMap = {}
  for item in items:
    key = item.entityType + ':' + item.entityId + ':' + item.operationType
    if key not in latestMap OR item.createdAt > latestMap[key].createdAt:
      latestMap[key] = item

  // Discard older duplicates (e.g. multiple edits of the same task)
  for item in items:
    if item NOT IN latestMap.values: markDone(item.localId)

  // Send only the latest version of each entity+operation
  for item in latestMap.values:
    markProcessing → processQueueItem → markDone OR markFailed(n+1)

Day Group Header Format

formatDayGroupLabel(dateStr):
  date = parseISO(dateStr)
  base = format(date, 'd MMM') + ' · ' + format(date, 'EEEE')
  if isToday(date):    return base + ' · Today'
  if isTomorrow(date): return base + ' · Tomorrow'
  return base
  // "16 Apr · Thursday · Today"
  // "17 Apr · Friday · Tomorrow"
  // "18 Apr · Saturday"

DnD Reorder + Reparent

handleDragEnd(active, over, delta):
  if not over OR active === over: return
  if delta.x > 50:
    // Drag-right → make dragged task a subtask of hovered task
    updateTask(active.id, { parent_id: over.id })
    scheduleFlush()
    return
  // Vertical drag → reorder
  reordered = arrayMove(localTasks, oldIndex, newIndex)
  for i, task in enumerate(reordered):
    updateTask(task.id, { sort_order: i × 10 })
  scheduleFlush()
  // sort_order: 0, 10, 20, … leaves room for future insertion
